# 한 장으로 끝내는 대한민국 제도 100

> 기업에는 비즈니스 모델이 있듯이, 국가에는 제도 모델이 있다.

대한민국의 주요 제도 **187개**를 법령·조직·절차·문서·기한으로 분해해, 제도마다 **한 장의 요약(캔버스)** 과 **업무구조도(스윔레인)** 로 읽을 수 있게 만든 공개 웹서비스입니다.

**👉 바로 보기: https://hosungseo.github.io/korea100/**

## 무엇을 볼 수 있나요

- **제도 찾기** — 홈에서 법령·기관·문서·병목 키워드로 187개 제도를 검색하고, 분야별로 훑어볼 수 있습니다. 최대 3개 제도를 같은 기준으로 나란히 비교할 수도 있습니다.
- **제도 한 장 요약** — 각 제도의 목적, 이해관계자, 법적 근거, 기관 권한, 돈·문서 흐름, 병목과 개선점을 9칸 캔버스 한 장으로 정리했습니다.
- **업무구조도** — 누가(행위주체 레인), 언제(단계), 무엇을(업무), 어떤 법조문에 근거해 하는지를 흐름도로 보여줍니다. 신청하는 국민, 심사하는 기관, 이의신청과 되돌아가는 경로까지 담습니다.
- **검증 대장 공개** — [/verification/](https://hosungseo.github.io/korea100/verification/)에서 "법령만으로는 확인할 수 없어 현장 확인이 필요한 항목" 582건을 사유와 함께 그대로 공개합니다.

## 이 데이터는 어떻게 검증되나요

정확성이 이 프로젝트의 핵심 가치입니다. 절차 노드 1,837개에 붙은 법령 인용 2,743건(법령·행정규칙·조약 294종)은 다음 원칙으로 관리됩니다.

1. **원문 대조** — 조문 인용은 국가법령정보센터의 현행 원문과 대조합니다. 조문 번호의 존재만이 아니라, 그 조문이 해당 업무의 행위·주체·기한을 실제로 규정하는지 확인합니다.
2. **다층 검증** — 기계 검사(구조·인용 형식·도달 가능성), 조문 의미 대조, 법령만으로 절차를 독립 재구성해 기존 구조도와 맞춰보는 교차 검증, 당사자(신청인·이의신청인) 시점의 워크스루를 거칩니다.
3. **지어내지 않기** — 원문을 확인하지 못한 인용(훈령·고시·내부 규정 등 공개 API 미제공 자료)은 조문 번호를 추정하지 않고 `unverified`로 표기해 현장 검증 대장으로 넘깁니다.
4. **정정 이력 공개** — 오류를 발견해 고친 내역은 각 제도 데이터의 `warnings`에 날짜·근거와 함께 남깁니다.

## 알아두실 것

- 구조도의 노드 상태(진행 중 등)는 실시간 행정 데이터가 아니라 **제도의 흐름을 설명하기 위한 편집 상태**입니다.
- 법령 기준일은 제도마다 `asOfDate`로 표시합니다. 법령은 개정되므로 중요한 판단 전에는 원문을 확인하세요.
- 이 콘텐츠는 제도 이해를 위한 참고 자료이며 **법률 자문이나 정부기관의 공식 해석을 대신하지 않습니다**.

## 이런 제도도 만들어 주세요

보고 싶은 제도가 없다면 [제작 요청 페이지](https://hosungseo.github.io/korea100/request/)에서 요청할 수 있습니다. 입력값은 서버에 저장되지 않고 사용자의 메일 앱 초안으로만 전달됩니다.
