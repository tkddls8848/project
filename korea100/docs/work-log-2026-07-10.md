## 13. 요청·지표·검증 운영 보완

- 요청 폼은 설정된 JSON API가 2xx를 반환한 경우에만 저장 완료로 표시한다.
- 수집 주소가 없거나 실패하면 입력 초안을 브라우저에 보존하고 이메일과 공개 GitHub 이슈 경로를 제공한다.
- 검색어 원문과 폼 내용을 제외한 최소 제품 이벤트를 선택형 분석 엔드포인트로 보낼 수 있게 했다.
- 100개 제도의 `fieldVerification`에서 448개 작업을 생성하는 현장 검증 큐와 `/verification/` 공개 대장을 추가했다.
- 현장 검증 항목은 제도, 분야, 검증 영역, 권장 근거, 안정적인 ID를 갖는다.
- 공식 법령 원문의 버전 서명을 레지스트리와 비교하는 `check:freshness` 모드와 주간 GitHub Actions를 추가했다.
- 변경이 발견되면 `law-freshness` 이슈를 만들거나 갱신하며, 저장소 secret `LAW_OC`가 없으면 쓰기 전에 실패한다.
