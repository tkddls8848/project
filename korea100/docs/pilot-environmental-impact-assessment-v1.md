# 환경영향평가 파일럿 v1

## 방향

`환경영향평가`는 제도 모델의 첫 기준 샘플로 쓰기 좋다. 법률, 대통령령, 시행규칙에 절차가 비교적 뚜렷하고, 행위주체도 사업자, 승인기관, 지자체, 주민, 협의기관, 검토기관으로 분리된다.

이번 파일럿은 단순 절차 설명이 아니라 `상태 인식형 업무구조도`로 만든다. 참고자료는 2026-07-09에 확인한 `html` 이메일의 `환경영향평가 상태 인식형 BPMN Swimlane` 목업이다.

## 한 장에서 보여줄 것

- 제도명: 환경영향평가
- 제도 유형: 협의ㆍ평가형
- 법령 기준일: 2025-10-23
- 핵심 법령: 환경영향평가법, 시행령, 시행규칙
- 대표 절차: 일반 환경영향평가 중 승인등을 받아야 하는 사업
- 현재 상태 예시: G2 조사ㆍ초안
- 현재 공 보유자 예시: 사업자ㆍ평가대행자
- 대표 병목 예시: 계절조사, 저감방안, 보완 요구

## 상태 인식형 구성

### 단계 게이트

| 게이트 | 단계 | 내용 |
| --- | --- | --- |
| G0 | 대상판정 | 대상사업 여부와 승인권자 확인 |
| G1 | 착수ㆍ스코핑 | 평가준비서, 평가항목ㆍ범위 결정 |
| G2 | 조사ㆍ초안 | 현황조사, 평가서 초안 작성 |
| G3 | 공고ㆍ공람 | 초안 접수, 공고, 공람 |
| G4 | 의견수렴 | 주민ㆍ관계기관 의견 수렴 |
| G5 | 본안ㆍ협의요청 | 평가서 작성, 협의 요청 |
| G6 | 검토ㆍ보완 | 협의기관 검토, 전문검토, 보완 |
| G7 | 협의통보ㆍ승인 | 협의내용 통보, 승인조건 반영 |
| G8 | 이행ㆍ사후조사 | 협의내용 이행, 사후환경영향조사 |

### 행위주체 lane

- 사업자ㆍ평가대행자
- 승인기관의 장
- 주관 시장ㆍ군수ㆍ구청장
- 주민 등
- 협의기관의 장
- 검토기관ㆍ전문가

### 상태 표시

- 완료: 이미 끝난 게이트
- 현재: 지금 공이 머무는 게이트와 주체
- 병목: 법정 절차상 대기, 보완, 조사 미완료, 의견 충돌
- 회귀: 보완 요구로 이전 단계로 돌아가는 흐름
- 현장검증 필요: 법령에는 없지만 실제 운영자료로 채워야 하는 처리시간, 시스템 입력, 내부 결재

## 법령 근거 연결

- 평가항목ㆍ범위 결정: 법 제24조, 시행령 제32조, 시행규칙 제8조
- 평가서 초안 작성: 법 제25조, 시행령 제34조
- 공고ㆍ공람과 주민 의견 수렴: 법 제25조, 시행령 제36조ㆍ제38조, 시행규칙 제9조
- 평가서 작성과 협의 요청: 법 제27조, 시행령 제46조ㆍ제47조
- 검토ㆍ보완ㆍ조정: 법 제28조, 시행령 제48조
- 협의내용 통보: 법 제29조, 시행령 제50조
- 협의내용 반영과 반영결과 통보: 법 제30조, 시행령 제51조, 시행규칙 제13조
- 이행과 사후환경영향조사: 법 제35조ㆍ제36조, 시행규칙 제19조

## 다음 편집 원칙

- 요약 화면에는 지나치게 넓은 인터랙티브 보드를 그대로 넣지 않는다.
- 요약은 `읽히는 한 장`, 상세는 `탐색하는 화면`으로 역할을 분리한다.
- 그림의 첫 시선은 현재 상태가 아니라 제도 전체 구조로 가야 한다.
- 병목은 주장으로 쓰지 않고 `어떤 단계에서 무엇이 대기 중인지`로 쓴다.
- 법령에 없는 처리시간과 실제 시스템 단계는 `현장검증 필요`로 표시한다.
