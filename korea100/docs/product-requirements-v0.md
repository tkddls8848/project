# 제품 요구사항 문서 v0

> 이 문서는 첫 10개 공개를 정의한 역사적 MVP 기록이다. 현재 100개 공개 서비스의 요구사항은 `docs/product-requirements-v1.md`를 따른다.

## 제품명

한 장으로 끝내는 대한민국 제도 100

## 목적

대한민국의 주요 제도를 법령, 조직, 절차, 예산, 문서, 데이터, 권한 관계까지 한 장의 구조도로 보여주는 온라인 제도 정보 서비스를 만든다.

첫 단계에서는 대표 제도 10개를 공개해 정보 구조와 시각화 포맷을 검증하고, 이후 같은 데이터 계약으로 100개까지 확장한다.

## 배경

한국의 제도는 법률, 대통령령, 시행규칙, 고시, 부처 직제, 예산사업, 신청 서식, 정보시스템이 결합되어 작동한다. 그러나 사용자는 제도가 어떤 기관과 문서, 예산, 절차를 통해 움직이는지 한눈에 파악하기 어렵다.

이 서비스는 제도를 텍스트 해설이 아니라 반복 가능한 `제도 모델`로 보여준다. 기업에 비즈니스 모델이 있듯이, 국가에는 제도 모델이 있다는 관점이 핵심이다.

## 대상 사용자

- 정책 이슈를 빠르게 구조화해서 이해하려는 시민과 학습자
- 신규 공무원, 공시생, 행정학·정책학 학생
- 국회·지방의회 보좌진과 정책 담당자
- 정책연구자, 컨설턴트, 공공기관 실무자
- 기자, 시민단체, 법학 학습자

## 사용자 문제

- 제도 설명이 법령 조문, 보도자료, 해설 자료로 흩어져 있다.
- 어느 기관이 결정하고 집행하는지, 어떤 문서와 예산이 오가는지 파악하기 어렵다.
- 법령상 절차와 실제 업무 흐름의 차이를 구분하기 어렵다.
- 비슷한 제도를 같은 기준으로 비교하기 어렵다.
- 반복 가능한 데이터·시각화 포맷이 필요하다.

## 제품 가설

대표 제도를 한 장 구조도와 상태 인식형 업무구조도로 공개하면 사용자는 제도 이해의 효용을 빠르게 확인하고, 다음 제작 대상과 개선 의견을 제안할 수 있다.

## 성공 기준

- 첫 10개 제도가 같은 포맷으로 공개된다.
- 환경영향평가가 상태 인식형 업무구조도의 기준 사례로 작동한다.
- 근거 법령, 핵심 행위자, 절차, 산출물과 병목을 한 화면에서 파악할 수 있다.
- 다음 제도 제작 요청을 메일 앱으로 보낼 수 있다.
- 방문, 검색, 비교와 피드백을 바탕으로 온라인 서비스를 개선할 수 있다.

## MVP 범위

### 포함

- 홈페이지와 첫 10개 제도 목록
- 제도별 한 장 요약과 상세 페이지
- 상태 인식형 업무구조도
- 제도 제작 요청 메일 초안
- 법령 기준일과 검증 상태
- 모바일 대응

### 제외

- 회원가입과 로그인
- 결제
- 관리자 페이지
- 완전 자동 법령 분석
- 법률 자문 또는 공식 해석
- 신청자 정보의 서버·브라우저 저장

## 첫 공개 제도 10개

1. 환경영향평가
2. 예비타당성조사
3. 재건축·재개발 정비사업
4. 지방소멸대응기금
5. 정보공개청구
6. 행정심판
7. 국민기초생활보장
8. 공공조달·국가계약
9. 국민건강보험
10. 개인정보 영향평가

## 핵심 기능 요구사항

### 카탈로그

- 제도를 우선순위와 유형별로 보여준다.
- 제도명, 법령, 기관, 문서와 절차 용어를 검색할 수 있다.
- 같은 기준으로 여러 제도를 비교할 수 있다.

### 제도 상세

- 제도명, 한 줄 요약, 목적, 법령, 권한, 절차, 돈·문서 흐름을 제공한다.
- 병목, 개선점, 관련 제도와 법령 기준일을 표시한다.
- 법령상 구조와 현장 검증 필요 항목을 구분한다.

### 업무구조도

- 단계 게이트와 행위자 레인을 보여준다.
- 업무 노드, 산출물, 다음 게이트, 병목과 회귀 흐름을 표시한다.
- 선택한 노드의 법적 근거와 검증 상태를 확인할 수 있다.

### 제작 요청

- 알고 싶은 제도명, 이유, 사용자 유형, 헷갈리는 지점을 입력한다.
- 입력값은 메일 초안을 만드는 데만 사용한다.
- 사이트는 요청 내용이나 연락처를 저장하지 않는다.

## 비기능 요구사항

- 정적 웹사이트로 운영할 수 있어야 한다.
- 모바일에서 문서 가로 넘침 없이 읽을 수 있어야 한다.
- 검색 유입을 위해 제도명, 절차명, 법령명을 텍스트로 노출한다.
- 새 제도를 추가할 때 기존 데이터 계약과 화면을 반복할 수 있어야 한다.

## 검증 지표

- 방문자 수와 제도별 페이지 체류 시간
- 검색 결과 0건 비율과 자주 사용하는 필터
- 비교와 상세 페이지 이용량
- 반복 요청되는 제도명
- 피드백에서 반복되는 혼란 지점

## 확장 단계

1. 정적 MVP와 환경영향평가 기준 샘플
2. 첫 10개 제도 공개
3. 검색·비교·검증 대장 운영
4. 동일한 데이터 계약으로 100개 제도 확장
