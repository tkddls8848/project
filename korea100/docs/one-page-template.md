# 제도 한 장 템플릿 v0

## 9칸 제도 모델 캔버스

| 칸 | 질문 | 내용 |
| --- | --- | --- |
| 1 | 이 제도는 무엇을 해결하나 | 목적, 문제상황, 정책목표 |
| 2 | 누구에게 영향을 주나 | 수혜자, 신청자, 규제대상, 비용부담자 |
| 3 | 법적 근거는 무엇인가 | 법률, 대통령령, 시행규칙, 조례, 고시 |
| 4 | 누가 권한을 갖나 | 중앙부처, 지자체, 위원회, 공공기관 |
| 5 | 절차는 어떻게 흐르나 | 신청, 접수, 심사, 협의, 결정, 통보, 집행 |
| 6 | 돈은 어떻게 흐르나 | 재원, 부담금, 보험료, 보조금, 교부금, 수수료 |
| 7 | 문서와 데이터는 어떻게 흐르나 | 신청서, 평가서, 심의자료, 통보서, 정보시스템 |
| 8 | 어디서 막히나 | 병목, 권한 충돌, 지연, 사각지대, 집행 부담 |
| 9 | 어떻게 바꿀 수 있나 | 개혁 포인트, 비교 제도, 개선 아이디어 |

## 한 장 레이아웃

### 상단

- 제도명
- 한 줄 요약
- 제도 유형
- 기준일
- 근거 법령

### 중앙

- 절차 swimlane 또는 제도 구조도
- 행위자: 신청자/수혜자, 집행기관, 심사기관, 재정기관, 감독기관
- 단계: 시작 조건, 제출, 심사, 협의, 결정, 통보, 이행, 사후관리
- 복잡한 절차형 제도는 `상태 인식형 업무구조도`로 확장한다.
  - 단계 게이트: G0, G1, G2처럼 현재 절차 위치를 표시한다.
  - 현재 공 보유자: 지금 일을 쥐고 있는 주체를 표시한다.
  - 병목: 지연 원인, 보완 필요, 대기 중인 산출물을 표시한다.
  - 회귀 루프: 보완 요구나 재협의처럼 앞 단계로 돌아가는 흐름을 표시한다.
  - 법령상 절차와 현장 추정값을 색상/라벨로 구분한다.

### 하단

- 돈의 흐름
- 문서/데이터 흐름
- 병목과 쟁점
- 개혁 포인트
- 관련 제도

## 작성 규칙

- 한 제도는 한 장에서 끝낸다.
- 법령명과 조문번호를 반드시 붙인다.
- 법령상 구조와 실제 운영 추정은 구분한다.
- 절차가 복잡하면 요약에는 대표 절차만 그리고, 예외는 상세 화면으로 분리한다.
- 문장은 짧게 쓴다.
- 이용자가 그림만 보고도 `누가, 무엇을, 어떤 근거로, 어떤 순서로` 하는지 알 수 있어야 한다.
- 절차형 제도는 `현재 위치`, `공 보유자`, `병목`, `다음 게이트`를 반드시 검토한다.
- 기본 화면은 정적인 한 장으로 만들되, 상세 화면은 상태 인식형 HTML/BPMN 구조로 확장할 수 있게 설계한다.
