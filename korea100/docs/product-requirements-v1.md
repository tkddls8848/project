# 제품 요구사항 문서 v1

기준일: 2026-07-10

## 제품 목표

대한민국 주요 제도를 법령, 권한, 기관, 업무 절차, 문서, 예산, 병목과 개선 관점에서 빠르게 찾고 비교하고 공유할 수 있는 공개 참고 서비스를 운영한다. 법령에서 확인한 사실, 범위를 더 정해야 하는 출처, 현장 확인이 필요한 운영 사실을 서로 다른 상태로 표시한다.

## 핵심 사용자

- 제도 구조를 빠르게 파악해야 하는 공무원·공공기관 실무자
- 국회·지방의회 보좌진, 정책연구자, 컨설턴트, 기자
- 행정학·정책학·법학 학습자와 일반 이용자

## 공개 범위

- 정규화된 제도 모델 101개와 상세 페이지 101개
- 업무 노드 1,594개와 법적 근거·검증 상태
- 공식 원문 연결 437건, 명시 조문 확인 3,767건
- 현장 검증 큐 448건
- 검색, 필터, 정렬, 최대 3개 비교
- 상세 한 장 요약, 핵심/전체 업무구조도, 노드 선택
- 메일 앱 기반 제도 제작 요청

## 기능 요구사항

### 카탈로그

- 최초 HTML에는 목록 표시에 필요한 경량 요약만 포함한다.
- 제도명, 법령명, 기관, 문서, 산출물, 병목, 개선점과 절차 용어를 검색한다.
- 분야, 유형, 검증 상태, 우선순위/가나다순을 조합할 수 있다.
- 검색·필터·정렬·비교 상태를 URL로 복원하고 공유한다.
- 최초 12개를 표시하고 사용자가 추가 항목을 요청할 때 확장한다.

### 비교

- 2개 이상 3개 이하 제도를 선택한다.
- 동일한 10개 기준으로 차이를 읽을 수 있어야 한다.
- 모바일에서는 표 영역만 가로 스크롤하고 문서 폭은 고정한다.

### 상세와 업무구조도

- 제도 목적, 이해관계자, 법령, 기관 권한, 대표 절차, 돈·문서 흐름, 병목, 개선, 관련 제도를 제공한다.
- 기본 모드는 핵심 흐름이며 전체 구조도로 전환할 수 있다.
- 전체 모드와 선택한 노드는 URL로 복원한다.
- 노드 상세에서 행위자, 산출물, 기한, 병목, 법적 근거, 공식 원문과 검증 상태를 확인한다.
- 데스크톱과 모바일에서 제도 전체 흐름과 선택 노드 상세를 확인한다.

### 검증과 신뢰

- `조문 확인`, `원문 연결`, `범위 확인`, `현장 검증`의 의미를 공개한다.
- 현장 검증 항목은 검색·분류할 수 있고 안정적인 ID를 가진다.
- 공개 근거 제보는 개인 정보와 비공개 내부 자료를 받지 않는다고 고지한다.
- 모든 상세 페이지는 법령 기준일과 비자문 고지를 표시한다.

### 요청과 지표

- 요청 입력값은 페이지가 열린 동안 메모리에만 유지한다.
- 요청 내용은 `mailto:` 초안을 만드는 데만 사용하며 서버나 브라우저 저장소에 기록하지 않는다.
- 연락처 입력란과 별도 구독 신청란을 제공하지 않는다.
- 이용 이벤트는 검색 결과 수, 필터, 비교, 상세 열기, 모드처럼 제품 개선에 필요한 최소 정보만 보낸다.
- 검색어 원문이나 입력 폼 내용은 분석 이벤트에 포함하지 않는다.

### 최신성

- 공식 출처 레지스트리와 현행 원문의 버전 서명을 주 1회 비교할 수 있다.
- 변경이 감지되면 공개 이슈를 생성하거나 기존 최신성 이슈를 갱신한다.
- 인증값은 저장소 파일에 기록하지 않고 GitHub secret으로만 제공한다.

## 비기능 요구사항

- GitHub Pages 정적 내보내기와 base path에서 모든 경로·자산이 동작한다.
- 모바일 390px에서 문서 가로 넘침이 없어야 한다.
- 키보드로 검색, 필터, 비교, 모드와 노드 기능을 사용할 수 있다.
- 공유 메타데이터, canonical URL, sitemap, robots, 구조화 데이터를 제공한다.
- 홈 정적 HTML은 상세 데이터 전체를 포함하지 않는다.
- 데이터 검증, 파서 테스트, ESLint, 보안 감사, 프로덕션 빌드를 통과한다.

## 운영 수용 기준

- manifest에 등록된 모든 상세 경로가 정적 생성된다.
- 홈 검색은 카드에 보이지 않는 절차·병목 용어도 찾는다.
- URL 재접속 후 검색, 비교, 업무구조도 모드와 노드가 복원된다.
- 요청 페이지는 입력값을 저장하거나 서버로 전송하지 않는다.
- 법령 변경 감시는 secret 누락 시 명확히 실패하고 데이터를 쓰지 않는다.
- 현장 검증 큐의 항목 수, ID, 제도 참조를 데이터 검증이 확인한다.
- 데스크톱과 모바일 브라우저에서 치명적 콘솔 오류와 문서 가로 넘침이 없다.

## 제외 범위

- 회원가입, 결제, 개인별 사건 관리
- 실시간 행정 사건 진행 상태
- 법률 자문, 공식 유권해석, 개별 사건 적용 판단
- 비공개 내부 자료 수집과 개인 정보가 포함된 현장 제보

## 성공 지표

- 검색 결과 0건 비율과 많이 사용되는 필터
- 상세 페이지 진입과 업무구조도 전체 모드 전환
- 비교 시작·완료와 업무구조도 이용
- 반복 요청되는 제도명과 직접 피드백
- 현장 검증 근거 제보와 최신성 이슈 해결 건수
