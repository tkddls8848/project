## 1. 이 세션에서 완료한 것

### 검증 캠페인 (L0~L4 전량 실행)

- 업무구조도 오류 검증 방법론을 설계하고 전 제도에 실전 적용했다. 인용·그래프·레인 오류 238건과 블라인드 재구성(L2) 정정 약 130건을 처리했다 (PR #27, #30, #39~#44).
- 방법론 정본: [verification-methodology.md](verification-methodology.md) — 오류 유형학 A~G, L0~L4 깔때기, 판정 원칙, 조례 예시 정책, 병렬 실행 교훈까지 모두 담겨 있다. **다음 세션에서 검증 작업을 이어갈 때 이 문서부터 읽는다.**
- L0 기계 감사 도구 신설: `web/scripts/audit-process.mjs` → `docs/audits/process-audit-<날짜>.json`
- 실행 레시피 4종: `docs/recipes/` (l1-citation-verification, l2-blind-reconstruction, admrul-verification, institution-creation)

### 정부조직 개편 부처명 반영

- 정부조직법 현행(2026-01-02 시행, MST 276049) 기준으로 전 제도 170건 수정: 기획재정부→기획예산처(예산·기금·예타)·재정경제부(세제·국고·공공기관관리, 국세청·관세청·조달청 소속), 기후에너지환경부, 산업통상부, 성평등가족부, 국가데이터처, 지식재산처.

### 신규 제도 13종 (109→123개, 전부 full)

- 인사혁신처 공무원임용령 + 행안부 지방인사제도과 계열 11종 제작.
- 외부 AI가 추가한 122(국가 조기승진제)·123(지방 특별승진)을 2회에 걸쳐 감사·보완 (PR #49, #50):
  - 122: 국가공무원법 §32①(장관 제청→인사처장 협의→총리 경유→대통령 임용) 반영, 처분성 정밀화, L4 = 인사혁신처 공고 제2026-375호(2026년 100명 선발, 접수 7/27~8/6).
  - 123: 특별승진 **취소 명문 규정 부재 확정** 기록(지공법 §39의3·국공법 §40의4·양 임용령 전수 대조, §65의2는 채용비위 전용), L4 = 지방공무원 임용령 2026-06-30 개정(인사교류·민간전담 경로 신설) 언론 보도.
- **반복 패턴 등록**: "5급 이상 임용 노드에 §32① 대통령 임용 체계 누락" — senior-civil-service와 122에서 재발. 인사 제도 만들 때 체크포인트.

### 법제처 DRF API 검증 인프라 확장

- `admrul`(행정규칙)·`ordin`(자치법규)·`trty`(조약)·`pi`(공공기관 규정) 타깃 확립. 사용법·타깃 지도는 [development.md](development.md).
- **ordin 함정 두 가지**: 본문 조회는 ID가 아니라 **MST 파라미터**여야 함, 조 태그는 속성이 붙으므로 `<조[ >]`로 파싱.
- 조례 예시 정책: 예시 지자체 1곳 원문 검증 + "— 예시" + "지자체별 상이" 명시, **서울은 되도록 제외(부산 기본, 경기 폴백)** — 사용자 지시.
- 검증에 쓴 조례 일련번호: 부산 주민참여예산 운영 조례 1949201, 부산 주민감사청구 조례 1697739(§2=300명), 부산 도시·주거환경정비 조례 2079585(§38), 경기도의회 회의규칙 2076913(§25의2), 경기도 조례규칙심의회 규칙 2072089.

## 3. 운영 교훈 (병렬 에이전트)

- 파이프(`| tail`)가 exit code를 삼킨다 — `> /tmp/v.log 2>&1; echo $?`로 직접 확인.
- 서브에이전트 최종 보고는 자주 유실된다 — 커밋 로그·데이터 실측으로 검수.
- `git add`는 자기 파일만 지정(광역 add가 타 에이전트 작업을 휩쓺), index.lock은 2초 재시도.
- `generate-field-verification-queue.mjs`는 **web/ 디렉토리에서 실행** 필수.
