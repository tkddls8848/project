5.3.1 적재 범위
데모 시드 쿼리 3종 중 절차형만 그래프 관계가 필요하다. 단일형/비교형은 그래프 없이 동작한다. 따라서 절차형 시연에 필요한 최소 노드와 엣지만 적재한다.
원칙:

전체 91건 서비스를 다 넣지 않는다
시드 시나리오 노드 5~10개만 적재
관계는 시연 흐름에 맞게 직접 설계
모든 데모 노드에 is_demo: true 플래그 부여

10. 리스크 및 대응
리스크영향대응Ollama gemma4:e4b 모델이 환경에 없음백엔드 부팅 불가Phase 0 환경 점검에서 ollama pull 강제, README 가이드시드 시나리오 서비스가 실재 카탈로그에 없음데모 외 시연 신뢰성 저하"가짜 노드는 시연 전용"임을 README 명시, 시연 외 쿼리는 fallback flow로 처리Neo4j 연결 정보 환경변수 누락시드 스크립트 실패.env.example에 변수 추가, README에 설정 가이드운영 데이터와 데모 데이터 혼재향후 충돌모든 데모 노드 is_demo: true 강제, 삭제 스크립트 동봉React Flow 첫 도입 학습 시간Phase 2 일정 지연Phase 1 진행 중 병렬로 별도 미니 데모 학습 권장단계 사이 시간이 짧아 인지 못함데모 임팩트 감소각 단계 최소 150ms 지연 강제, 시각적 펄스 충분히 길게LLM 응답 지연으로 토큰 스트림 끊김데모 인상 저하Ollama 모델을 가벼운 것으로 교체 가능하게 환경변수 분리X-API-Key 누락 시 데모 페이지 401시연 사고lib/streamClient.ts에서 자동 주입시연 외 절차형 쿼리에서 flow 빈약데모 외 사용 시 UX 저하"데모 시드는 운전면허 시나리오 한정"이라고 명시envelope 스키마 변경 시 양쪽 동기화리팩토링 비용types.ts에 단일 진실 정의, 백엔드 Pydantic 모델과 수동 동기화 (Phase 3 이후 자동화 검토)
